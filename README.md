CS 255 System Analysis and Design Portfolio

Reynaldo Williams

Project Summary

The DriverPass project involved designing a system for a client who wanted to improve the process of helping students prepare for their driving exams. The client required a web-based system that allowed students to register, purchase packages, schedule driving lessons, and take practice tests aligned with DMV requirements. The goal was to create a structured, secure, and user-friendly system that improved scheduling efficiency and learning support.

What I Did Well

I believe I did particularly well in translating business requirements into structured system models. My UML diagrams—including use case, activity, sequence, and class diagrams—clearly mapped user interactions and system behavior. I also ensured that security, maintenance, and user accessibility were consistently addressed throughout the design.

What I Would Improve

If I were to revise part of this project, I would further refine alternative and exception flows in my diagrams to represent more real-world edge cases. Adding additional validation paths and expanded error-handling scenarios would strengthen the overall robustness of the design.

Interpreting User Needs

I interpreted the user’s needs by focusing on clarity, usability, and accessibility. The client emphasized structured scheduling, updated practice test content, and administrative oversight. I translated these needs into functional system components and ensured that user roles were clearly defined. Considering user needs is critical because a technically strong system can still fail if it is not intuitive or aligned with real-world workflows.

My Approach to Designing Software

When designing software, I begin with requirement analysis and stakeholder understanding before modeling system behavior. I use UML diagrams to visualize structure and interactions, which helps ensure completeness and logical consistency. In the future, I will continue using structured documentation, iterative refinement, and user-focused thinking to analyze and design scalable systems.
